# Contributing

This repo is the public source of truth for OpenHands Champions. The way you become “official” for a season is by opening a PR that adds your entry.

## Before you open a PR

- Your entry must be **publicly verifiable**.
- You must open the PR **yourself** (Champions are volunteers; the OpenHands team reviews/merges, but doesn’t add people on their behalf).

## Add yourself to the list

1. Edit `README.md`.
2. Add a new row in the correct table:
   - **2026 (Season 1) — Hackers**, and/or
   - **2026 (Season 1) — Insiders**
3. Prefer placing your row **alphabetically by handle** within the table to keep it easy to scan.

### Hacker row format

Required:
- Project name
- 1–2 sentence description
- Links:
  - project repo link (if applicable)
  - blog post / write-up
  - demo (live or recorded)

Suggested `Links` cell format:

`[repo](...) · [write-up](...) · [demo](...)`

Suggested `Badge` cell format:

`![Hacker 2026](badges/2026/openhands-hacker-2026.svg)`

### Insider row format

Insiders are listed publicly once participation is verified, but **attendance details remain private**.

Suggested `Badge` cell format:

`![Insider 2026](badges/2026/openhands-insider-2026.svg)`

## PR checklist

Your PR should:
- [ ] Add your row(s) without changing other entries (unless fixing obvious typos)
- [ ] Use links that are public and stable
- [ ] Match the table column format
- [ ] Include all required links for the Hacker track

## Questions / changes

If you want to propose changes to the program structure or table format, open an issue or start a discussion (depending on what’s enabled for this repo).
